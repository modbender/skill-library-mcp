**追加した機能**
- `--save-auth` で認証情報を保存
- 保存先デフォルト: `~/.config/calibre-metadata-apply/auth.json`
- 既定保存内容: `username` + `password_env`（平文パスワードは保存しない）
- 必要なら `--save-plain-password` で平文保存も可
- 次回以降は `--username` / `--password-env` 省略で自動読込
- dry-run 出力コマンド上の `--password` をマスク（漏えい防止）

**使い方（初回だけ）**
```bash
cat references/changes.example.jsonl | python3 skills/calibre-metadata-apply/scripts/calibredb_apply.py \
  --with-library "http://HOST:PORT/#LIBRARY_ID" \
  --username "your_user" --password-env CALIBRE_PASSWORD \
  --save-auth
```

その後は普通に:
```bash
cat references/changes.example.jsonl | python3 skills/calibre-metadata-apply/scripts/calibredb_apply.py \
  --with-library "http://HOST:PORT/#LIBRARY_ID"
```

必要なら次に、`calibre-catalog-read` 側も同じ認証キャッシュ仕様に完全統一して、検索→確認→適用を全部ノンストップに揃える。
